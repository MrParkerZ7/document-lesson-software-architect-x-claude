## 7.2 T-Shaped Learning

### The T-Shaped Professional

```
┌─────────────────────────────────────────────────────────────┐
│                 T-Shaped Learning Model                      │
│                                                              │
│  ──────────────────────────────────────────────────────     │
│  │                    BREADTH                          │     │
│  │  Cloud │ Security │ Data │ DevOps │ Networking │   │     │
│  │  ──────┼──────────┼──────┼────────┼────────────┼   │     │
│  │   ✓    │    ✓     │  ✓   │   ✓    │     ✓      │   │     │
│  ──────────────────────────────────────────────────────     │
│                          │                                  │
│                          │  DEPTH                           │
│                          │                                  │
│                          │  Your Primary                    │
│                          │  Specialization                  │
│                          │                                  │
│                          │  • Expert-level                  │
│                          │    knowledge                     │
│                          │  • Go-to person                  │

│                          │    recognized                    │
│                          │                                  │

### Breadth vs Depth by Career Stage

| Stage | Breadth | Depth | Strategy |
|-------|---------|-------|----------|
| **Junior** | High | Medium | Explore, find interests |
| **Mid** | Medium | High | Specialize, build expertise |
| **Senior** | High | High | Strategic breadth, maintained depth |
| **Principal+** | Very High | Selective | Systems thinking, strategic focus |

## 7.3 Learning Cadence

### Daily Learning

| Activity | Time | Examples |
|----------|------|----------|
| **News scanning** | 15 min | Newsletters, RSS feeds, Twitter |
| **Reading** | 15-30 min | Articles, documentation |
| **Problem solving** | Ongoing | Work challenges as learning |

**Recommended Newsletters**:
- The Morning Paper (academic papers)
- TLDR (tech news)
- Bytes (web development)
- DevOps Weekly
- Architecture Weekly

---

### Weekly Learning

| Activity | Time | Examples |
|----------|------|----------|
| **Deep reading** | 2-3 hours | Technical blogs, research |
| **Hands-on practice** | 2-3 hours | Side projects, labs |
| **Podcast/video** | 1-2 hours | Commute, exercise time |
| **Community engagement** | 1 hour | Forums, discussions |

**Recommended Podcasts**:
- Software Engineering Daily
- The Changelog
- Kubernetes Podcast
- AWS Podcast
- Software Architecture Radio

---

### Monthly Learning

| Activity | Time | Examples |
|----------|------|----------|
| **New tool exploration** | 4-8 hours | Try new technology |
| **Meetups** | 2-4 hours | Local community events |
| **Structured course progress** | 4-8 hours | Online courses |
| **Skill assessment** | 1 hour | Review learning progress |

```
┌─────────────────────────────────────────────────────────────┐
│              Learning Effectiveness Model                    │
│                                                              │
│  70% - EXPERIENTIAL                                         │
│  ─────────────────────                                      │
│  • On-the-job challenges                                    │
│  • Side projects                                            │
│  • Stretch assignments                                      │
│  • Production problems                                      │
│                                                              │
│  20% - SOCIAL                                               │
│  ─────────────────────                                      │
│  • Mentoring (giving and receiving)                         │
│  • Peer discussions                                         │
│  • Code reviews                                             │
│  • Architecture discussions                                 │
│                                                              │
│  10% - FORMAL                                               │
│  ─────────────────────                                      │

### Active Learning Techniques

| Technique | Description |
|-----------|-------------|

## 7.6 Managing Learning Overload

### Avoiding Learning FOMO

```
┌─────────────────────────────────────────────────────────────┐
│              Managing Technology FOMO                        │
│                                                              │
│  You can't learn everything - and you don't need to         │
│                                                              │
│  FOCUS FILTERS                                              │
│  ─────────────────                                          │
│  • Is this relevant to my current role?                     │
│  • Does this align with my career goals?                    │
│  • Is this trend durable or a fad?                          │
│  • Will this make me significantly better?                  │
│                                                              │
│  TIME ALLOCATION                                            │
│  ─────────────────                                          │
│  • 70% - Skills directly applicable to current work         │
│  • 20% - Skills for next career move                        │
│  • 10% - Exploration and experimentation                    │
│                                                              │
│  SAYING NO                                                  │
│  ─────────────────                                          │
│  • Not learning X doesn't make you a bad architect          │
│  • Deep expertise beats shallow breadth                     │
│  • You can learn later when/if it becomes relevant          │
└─────────────────────────────────────────────────────────────┘
```

### When to Learn Something New

| Signal | Action |
|--------|--------|
| **Direct work need** | Prioritize immediately |
| **Career goal alignment** | Add to learning plan |
| **Industry adoption growing** | Monitor, learn basics |
| **Interesting but niche** | Defer, revisit later |
| **Hype without substance** | Skip or surface awareness |
